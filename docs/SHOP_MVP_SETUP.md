# Shop MVP Setup

## 1) Environment Variables

Set these in local `.env.local` and in Vercel project settings:

- `NEXT_PUBLIC_SITE_URL`
- `MERCADO_PAGO_ACCESS_TOKEN`
- `MERCADO_PAGO_WEBHOOK_SECRET`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `RESEND_API_KEY`
- `RESEND_FROM_EMAIL`
- `CONTACT_RECEIVER_EMAIL`

## 2) Supabase Tables

Run this SQL in Supabase:

```sql
create table if not exists public.orders (
  id bigint generated by default as identity primary key,
  stripe_checkout_session_id text not null unique,
  stripe_payment_intent_id text,
  customer_email text,
  customer_name text,
  amount_total integer,
  currency text,
  payment_status text not null,
  order_status text not null default 'paid',
  locale text not null default 'de',
  created_at timestamptz not null default now()
);

create table if not exists public.order_items (
  id bigint generated by default as identity primary key,
  order_id bigint not null references public.orders(id) on delete cascade,
  book_slug text not null,
  book_title text not null,
  quantity integer not null default 1,
  unit_price integer,
  currency text,
  created_at timestamptz not null default now()
);
```

## 3) Mercado Pago Configuration

The current MVP checkout creates Mercado Pago preferences dynamically from
catalog data (`price`, `currency`, `checkoutEnabled`). For COP use numeric
strings such as:

- `price: "69000"`
- `currency: "COP"`

## 4) Mercado Pago Webhook

In Mercado Pago dashboard, create webhook endpoint:

- `https://<your-domain>/api/mercadopago/webhook?secret=<same-secret-as-env>`

Recommended events:

- Payment notifications (`payment`)

Set secret into:

- `MERCADO_PAGO_WEBHOOK_SECRET`
